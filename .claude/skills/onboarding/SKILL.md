# 创作者 Onboarding（建档案#N）

> 把"谁都能用"落地。引擎(选题/文案/复盘)读 `profiles/{id}/profile.json` 运转,
> 本 skill 负责把一个新创作者**问 + 抓**成那份档案。
> 字段定义见 `profiles/_schema/profile.schema.md`,北极星枚举见 `北极星枚举.md`。

## 铁律（优先级最高）

1. **必须闭环,不许问一半就停**。onboarding 的交付物不是"问了几个问题",是一份**用户确认过、校验通过、北极星已锁定**的 profile.json。没走到 Step 5 报出"建成"就是没做完,别停。
2. **北极星必须落成具体值,不能停在"待定"**。综合用户自述 + 数据反推后,给出**明确的指标 + 阈值 + 一句为什么**(如"涨粉率 > 1 + 完播率 ≥ 5%"),不能只说"看情况"。
3. **定稿前必须出'汇总确认卡'让用户过目**(Step 4)。逐项摆出北极星/受众/平台/红线/独家点,问"这样对吗、要改哪项",用户点头才写文件。别自己闷头写完就完事。
4. **一次只问一个问题**。像聊天一样一题一题来,等 TA 答完再问下一题;每题给 2-3 个提示/例子帮 TA 开口。别一次甩一堆问题,新手会懵。
5. **允许"不知道/答不清",这时你来兜**。用户答"不知道""没想好"是正常的——**别逼 TA 硬答,也别自己瞎填**。给具体选项让 TA 选、或换个更好答的问法、或举例引导;能推的字段可以先给个建议默认值让 TA 确认。实在定不了的项标"待补",不卡流程往下走(北极星除外,它必须定)。
6. **不替用户编关键答案**。北极星、受众、壁垒、红线填错整个引擎都歪——这些要 TA 亲口确认;你可以给建议,但最终值 TA 点头。
7. **能抓就别只问**。给了账号就**用真实近 30 条数据反推**,优先于用户嘴说的。两者冲突时把冲突摆出来、让用户拍板。
8. **写完必校验**。跑 `engine/validate_profile.py`,过了才算建成;改库(蒸馏套路)落库前给用户看。

## 流程

### Step 0 · 开场（别一上来问账号名）
一句话说清要干嘛,直接进第一个问题。账号名放最后再问(纯展示用,拿它+拼音生成 `id` 做目录)。
> 开场例:"来,我问你几个问题就能给你配一套专属的内容助手。一题一题来,答不上来也没关系,我帮你想。第一个——"

### Step 1 · 逐题问诊（一次一题,每题带提示,答不清你来兜）

**每题的模板**:先抛主问句 → 跟 2-3 个💡提示/例子 → 用户答不出就用🤷兜底话术。收到答案先复述确认再进下一题。

**Q1 · 你想做什么号,最想要的结果是什么?**(定北极星,最关键)
- 💡 常见目标:涨粉做大 / 带货卖东西 / 导私域(加微信·进群·接咨询)/ 接广告 / 立人设。
- 🤷 答不清:"换个问法——你做号做成了,最想看到的是账号涨了很多粉,还是真金白银卖出去东西,还是有人主动来找你?" 拿到目标后**查 `北极星枚举.md` 目标→北极星映射**给推荐指标+阈值,让 TA 确认(别直接套迪迪的"涨粉率+完播")。
- 填:`north_star` `anti_metrics` `positioning` `monetization` `content_pillars`

**Q2 · 你想让谁看到你的内容?有没有明确不想要的人?**
- 💡 说清楚点:年龄段+身份+痛点。例:"20-30岁想跳槽涨薪的职场女生""预算不高的学生党宝妈"。
- 🤷 答不清:"那你想象一个最典型的观众,TA 几岁、在忙什么、有什么烦恼?" 排除人群没有可留空。
- 填:`audience.want` `.exclude`

**Q3 · 你有什么是同赛道大部分人没有的?(经历/职业/资源/踩过的坑)**
- 💡 例:"我做过两年美妆柜姐,知道柜台话术的坑""我真用这套方法从0做到1万粉"。哪怕"我真花过三年"也算壁垒。
- 🤷 答不清:"你比刷同类内容的人多知道、多做过什么?一句大白话就行。" 实在没有 → 标"待挖掘",后面选题时再帮 TA 找。
- 填:`persona.moat`

**Q4 · 主战哪个平台?发完想把人往哪引?**
- 💡 平台:抖音 / 小红书 / TikTok / 视频号。承接:粉丝群、微信、评论区口令、link in bio、暂时不导。
- 🤷 答不清:给默认"主平台 + 暂不导私域,先做内容",让 TA 确认。
- 填:`platforms`(决定限流规则模块 + 自动取数是否支持——目前仅抖音全自动)

**Q5 · 有没有 3 个你刷到会停下来、甚至有点嫉妒的同赛道账号?**
- 💡 就是你心里"想做成 TA 那样"的人;报平台+名字即可。
- 🤷 想不出:可以现在一起搜、或先给1个,不卡流程;后面用 blogger-distiller 蒸馏成你的套路库弹药。
- 填:`benchmark_accounts`(→ Step 2B 自动蒸馏)

**Q6 · 能发我 1-2 条你自己比较满意的旧文案/口播吗?**(为了让写出来像你,不像 AI)
- 💡 贴正文或文件路径都行;我会提炼你的说话习惯(爱用的词、语速、开头方式)。
- 🤷 新号还没有:"那没关系,先按你平时说话的感觉来,写文案时我们再校准。" 标 `tone_samples` 待补。
- 填:`tone_samples`

**Q7 · 视觉上喜欢什么感觉?封面要不要出镜?**
- 💡 给选项挑:奶油编辑撞色 / 小清新 / 暗黑科技 / 极简手绘;配色偏好;真人出镜 or 只放产品/画面。
- 🤷 没想法:给 2-3 个模板缩略描述让 TA 选一个,别让 TA 从零想。
- 填:`visual_style`

**最后 · 这个号叫什么名字?** 纯展示 + 生成目录 id 用,轻描淡写问一句即可。
- 填:`name`,派生 `id`(英文/拼音短名)。

### Step 2 · 自动抓（关键升级,别省）

**A. 反推北极星(自动取数,一步到位)** —— 跑 `engine/fetch/pull.py`
- **用户只要在浏览器(Chrome/Edge)登录抖音**,一条命令自动拿全量底表反推北极星:
  ```bash
  .venv/bin/python engine/fetch/pull.py onboarding {id} --goal {商业目标}
  ```
- 自动读浏览器 cookie(`get_cookie.py`)→ 自包含取数(`fetch_backend.py`,含完播/涨粉/留存全量底表)→ `baseline.py` 算基线 + 反推,**自动标"自述 vs 数据"冲突**(铁律2:数据优先,让用户拍板)。
- 首次读 Chrome 可能弹一次钥匙串授权,点允许。没装依赖先 `pip install -r engine/fetch/requirements.txt`。
- 不依赖任何私有 repo;自己账号的完播/涨粉都能拿(走登录态,不是公开数据)。
- ⚠️ **自动取数目前只支持抖音**。主战小红书等其他平台:本步跳过、明说"基线待补",北极星按第1问+枚举表定;用户能贴后台数据则手动喂 `compute_baselines`。

**B. 对标蒸馏 → 种套路库(第5问的 3 个账号)**
- 对每个对标账号跑 `blogger-distiller`(`~/.claude/skills/blogger-distiller`,走 TikHub 公开 API 自动扒,不靠登录态)。
- 模式 A(拆对标),采集量默认 30。
- 蒸馏出的选题公式/钩子/标题模板 → 作该赛道的套路库样本(人工确认后落库)。
- ⚠️ TikHub 额度用尽或接口变 → 退回让用户手动贴内容。

**C. 自动挂信息源(选赛道后)**
- 据受众判赛道 → 填 `sources_track` → 挂赛道→信息源路由表(路由表完整版待建,先填赛道名)。

### Step 3 · 综合定北极星（把它"定下来"）
把用户自述(第1问)+ 数据反推(Step2A)合成,**给出明确的北极星**:指标 + 阈值 + 一句为什么。
- 例:"你说要涨粉,数据也支持,那定 **涨粉率 > 1**(受众匹配)+ **完播率 ≥ 5%**(底线)。"
- 若数据和自述打架(如自述涨粉、数据显示收藏强),**两个都摆出来,让用户选**,选完就锁。
- 顺带把其它字段的初值也想好(受众/平台/红线/独家/语气/视觉),不确定的留占位。

### Step 4 · 汇总确认卡（闭环关键,必做,别跳）
把整份档案**排版成一张卡**给用户过目,逐项列清、请 TA 确认或改:

```
【你的账号档案 · 待确认】
· 账号:{name}（id: {id}）
· 定位:{positioning}
· 北极星:{指标+阈值}  ← 为什么:{一句}
· 不看:{anti_metrics}
· 目标受众:{want}；排除:{exclude}
· 主战平台 / 私域承接:{platforms}
· 独家壁垒:{moat}
· 红线:{redlines}
· 视觉风格:{visual_style 或 待定}
· 对标账号:{3个，待蒸馏} ｜ 语气样本:{有/待补}
—— 这样对吗?要改哪项直接说;都 OK 我就锁定建档。
```
**用户明确说"可以/锁定/对"之前,不写文件**。用户改哪项就改哪项,改完再确认一遍。

### Step 5 · 确认后:写档 + 校验 + 报"建成"
用户点头后:
1. 按 `profile.schema.md` 写 `profiles/{id}/profile.json`(`presentation_rule` 默认"结果先行",不确定项留 `{"_待填":"..."}`)。
2. 校验(焊点,必跑,从仓库根目录):
   ```bash
   python3 engine/validate_profile.py {id}
   ```
   ❌ 硬错误(缺字段/指标不在引擎/阈值解析不了)→ 回去补,不算建成。⚠️ 警告(无对标/语气)→ 告知可后补。
3. (可选)拿样本数据跑一次复盘,确认按新北极星判定:`echo '<raw json>' | python3 engine/review.py {id}`。
4. **明确报出建成**:"✅ 档案建好了。你的号锁定北极星 = {指标+阈值};下一步可以直接『选题』。" ← 到这句才算 onboarding 完成。

---

## 产出物（缺一不可 = 没闭环）
1. 一张**用户确认过**的汇总确认卡(Step 4)。
2. `profiles/{id}/profile.json` — 校验通过的档案,**北极星已锁定成具体指标+阈值**。
3. 套路库种子(对标蒸馏,人工确认后落库)。
4. 收尾一句:锁定的北极星是啥、为什么(数据反推和嘴说冲突时讲清怎么定的)+ 指路下一步「选题」。

## 三层提醒（别越界）
- 只往 profile 填**第一层字段**(问出来的)。
- **第二层通用机制**(打分卡/钩子14招/2秒跳出/人味儿原则)不进 profile,引擎自带。
- **第三层样本**(套路库弹药)由对标蒸馏种入,不手抄迪迪的样本。
