# NorthStar Studio · 通用内容生产 agent

> 短视频内容生产流水线:**引擎共用,档案每人一份**。每个环节读 `profiles/{id}/profile.json` 运转,不写死任何创作者。
> 新用户第一件事:说"帮我建档案"走 onboarding,建成前别跳进选题/文案。

## 🔝 最高铁律（凌驾一切，冲突时以此为准）

1. **第一性原理思考与复盘**：从根本事实往上推，不靠类比、惯例、"别人都这么做"、现成结论。复盘归因必须问到根因（指标到底在测什么、机制上为什么），别停在表层模式匹配；数据与套路冲突时信数据、拆机制。
2. **省 token 优先**：结果先行、工具先行；表格/清单 > 大段文字；不复述、不铺垫、不输出没被要的东西。仅"写文案正文"例外（保人味）。

## 有哪些 skill(在 `.claude/skills/`,按用户说的话触发)

| skill | 用户说什么触发 | 干什么 |
|---|---|---|
| onboarding | "帮我建档案""配一下我的账号" | 8问+自动拉数据反推北极星 → 汇总确认卡 → 建 profile.json(必须闭环到"✅建成") |
| 选题 | "出选题""今天做什么内容" | 按档案赛道拉热点,五维打分排序 |
| 文案 | "写文案""写口播" | 先定口吻来源(自己/学博主/上传旧稿),再写口播稿+发布包 |
| 预审 | "预审""能发吗""会不会限流" | 三层红线扫+AI标注+资质门槛,红黄绿清单 |
| 复盘 | "复盘""这条表现怎么样" | 自动取底表,按该号北极星判命中/扑+归因 |

## 工作流与关键路径

- 全链 SOP:`工作流.md`(onboarding→选题→文案→预审→视频生产→复盘→闭环回选题)
- 档案层:`profiles/{id}/profile.json`,字段表在 `profiles/_schema/`;**任何环节动手前先读当前用户的档案**
- 通用机制:`共享机制/`(评分卡/钩子/人味儿写作/红线库/信息源路由表)——机制共用,别往里写个人化内容
- 引擎脚本:`engine/`(judge/review/baseline/validate) + `engine/fetch/`(自动cookie取抖音底表/pull一步取数/网页日报) + `engine/sources/`(热点源)
- 取数用项目 venv:`.venv/bin/python`(依赖见 `engine/fetch/requirements.txt`)

## 硬约束

- **没有档案不干活**:用户没建档就要选题/文案 → 先引导走 onboarding。
- **判定只认档案北极星**:不看播放量等 `anti_metrics`,不套用示例档案(迪迪/职场琳)的指标。
- **profile 改动必须过校验**:`python3 engine/validate_profile.py {id}`。
- 真实用户档案不提交 git(.gitignore 已配);cookie/底表数据只在本机。
