# NorthStar Studio · 内容生产 Agent(通用版)

> 一套跑在 Claude Code 里的短视频内容生产流水线:**选题 → 文案 → 预审+预测 → 出片 → 复盘**,外加自动拉你的抖音后台数据。
> 核心设计:**引擎共用,档案每人一份**。你回答几个问题建好自己的档案(北极星指标/受众/口吻/红线/视觉风格),之后每个环节都按你的档案运转——不是按别人的号硬套。

## 它能帮你做什么

| 环节 | 你得到什么 |
|---|---|
| 个性化配置 | 问答式建档 + 自动拉你近30条真实数据反推"你的号该看什么指标" |
| 选题 | 按你的赛道自动挂信息源,拉热点,五维打分排序的选题清单(不满意无限刷新) |
| 文案 | 口播稿 + 发布包(标题/tag/封面/预测卡);口风不对可蒸馏你想要的博主、或喂你自己的旧稿 |
| 预审+预测 | 发布前红黄绿风险单(限流词/AI标注/资质门槛,给最小改动)+ 数据预测 |
| 视频生产 | 按你的视觉风格出封面/幻灯片/编排表(模板库可挑可换) |
| 复盘 | 发布后自动取底表,按**你的**北极星判命中/扑,留存归因 + 限流预警 + 抖音日报 |
| 闭环 | 验证好的套路加权回你的套路库,下一轮选题更准 |

数据侧:**你只要在浏览器登录抖音**,agent 自动读取登录态拉创作者后台全量底表(完播率/涨粉/留存曲线/播放来源),不用手抠 cookie。

## 什么是"北极星指标"?

**北极星 = 你这个号唯一该盯的那 1-2 个指标**,由你的商业目标决定——所有环节(选题打分/预测/复盘判定)都对着它,其它数字(比如播放量)一概不看,防止被虚荣指标带偏。

| 你的目标 | 北极星该看 | 什么意思 |
|---|---|---|
| 涨粉/立人设 | **涨粉率**(涨粉÷千播) | 每一千个看到的人有几个关注你——受众准不准 |
| 带货 | **GMV/客单** | 这条视频归因了多少成交,播放再高不成交也没用 |
| 导私域(咨询/社群) | **加微率**(加微÷播放) | 评论区口令→加到微信的转化 |
| 做干货/工具号 | **收藏率**(收藏÷点赞) | >0.7 说明观众觉得"有用要留着"——精准信号 |
| 所有号的底线 | **完播率** | 先过合格线(如 5%),完播不行一切白搭 |

例:迪迪的号是 AI 工具号,北极星=涨粉率>1 + 完播率≥5%,明确**不看**播放量和总涨粉——因为一个精准的工具圈粉丝比一百个路人粉值钱。onboarding 时 agent 会拉你近 30 条真实数据,和你嘴上说的目标对比,冲突时摆给你看、你拍板。完整枚举和判定规则见 [profiles/_schema/北极星枚举.md](profiles/_schema/北极星枚举.md)。

## 建档时会问你什么(逐题问诊,不是填表)

个性化配置是**一场对话,一次只问一个问题**,每题给你提示和例子;**答不上来也没关系,agent 会给你选项、换个问法、或先给个建议默认值帮你想**,不逼你硬答。逐题:

| 问你什么 | 提示/答不清时 | 决定档案里的什么 |
|---|---|---|
| ① 你想做什么号,最想要的结果? | 涨粉/带货/导私域/接广/立人设;想不清就问"最想看到涨粉、卖货、还是有人来找你" | **北极星指标** + 定位 + 变现 |
| ② 想让谁看到?有没有明确不要的人? | 年龄+身份+痛点;答不清就想象一个最典型的观众 | 信息源 + 套路权重 |
| ③ 你有什么同赛道大部分人没有的?(经历/职业/坑) | "做过两年柜姐""真用了三年";实在没有标待挖掘 | 选题打分的"独家性"维度 |
| ④ 主战哪个平台?发完想把人往哪引? | 抖音/小红书/TikTok;承接=群/微信/link in bio,可先不导 | 限流规则 + 漏斗 |
| ⑤ 有没有 3 个你刷到会嫉妒的同赛道账号? | 想做成 TA 那样的人;想不出可现搜或后补 | 自动蒸馏 → 种套路库 |
| ⑥ 发我 1-2 条你满意的旧文案? | 让文案像你;新号没有就先按你说话感觉来,后补 | 语气样本 |
| ⑦ 视觉喜欢什么感觉?出不出镜? | 给模板选项挑,别从零想 | 封面/幻灯片风格 |
| (最后)号叫什么名字? | 纯展示 + 生成目录 | 档案 id |

问完不是结束:agent 会**综合你的回答定出带阈值的北极星**(如"涨粉率>1 + 完播率≥5%"),出一张**汇总确认卡**让你逐项核对、你点头才建档。给了抖音账号的话,还会**自动拉你近 30 条真实数据**反推——你说的目标和数据打架时,把两边摆出来让你拍板(数据往往比感觉准)。

## 安装

前置:macOS + [Claude Code](https://claude.com/claude-code) + Python 3.10+ + Chrome/Edge(登录着抖音)。

```bash
git clone https://github.com/weid00360-bot/northstar-studio.git
cd northstar-studio
python3 -m venv .venv
.venv/bin/pip install -r engine/fetch/requirements.txt
```

然后用 Claude Code 打开这个目录,对它说"帮我建档案"开始 onboarding。

> 首次自动读浏览器 cookie 时,macOS 会弹一次钥匙串授权("访问 Chrome Safe Storage")→ 点允许。

## 快速上手(三条命令验证装好了)

```bash
# 1. 校验示例档案(引擎能跑)
cd engine && python3 validate_profile.py 迪迪

# 2. 自动拉你的近30条底表,反推你的北极星(需浏览器登录抖音)
../.venv/bin/python fetch/pull.py onboarding 迪迪 --n 30 --goal 涨粉

# 3. 抖音日报:近10条按北极星判命中(命令行版)
../.venv/bin/python fetch/daily_report.py 迪迪 --n 10
```

跑通后,把"迪迪"换成你自己的档案 id(onboarding 会帮你建)。

想要**每早自动弹出的网页版日报**:
```bash
bash scripts/install_daily.sh install 你的id 8   # 每天 8:00 自动生成并打开浏览器
bash scripts/daily_run.sh 你的id                 # 立刻手动跑一次
```

## Agent 完整工作流

```
        ┌─────────────────────────────────────────────────────┐
        │  onboarding(一次性建档)                              │
        │  逐题问诊 + 自动拉近30条数据反推北极星               │
        │  → profiles/{你}/profile.json(校验通过才算建成)      │
        │  → 对标蒸馏种进你的套路库                             │
        └─────────────────────────────────────────────────────┘
                          │ 档案建好,下面每天跑
                          ▼
   ┌──► ① 选题   读你的档案 + 评分卡 + 赛道信息源路由表
   │       拉热点 → 九类爆款公式 × 你的独家壁垒 → 5维打分 → 清单(你拍板)
   │          │
   │          ▼
   │    ② 文案   读你的语气样本 + 钩子/结构/人味儿机制
   │       口播稿(对话里迭代,定稿才存)→ 发布包(标题/tag/封面/预测卡)
   │          │
   │          ▼
   │    ③ 预审+预测   三层红线扫(通用限流+平台官方+你的定位红线)
   │       红黄绿清单给最小改动 · AI标注/资质门槛 · 预测不满→回②改钩子
   │          │
   │          ▼
   │    ④ 视频生产   按你的视觉风格出封面/幻灯片/编排表 → 成片
   │          │
   │          ▼
   │    ⑤ 发布(你手动)
   │          │
   │          ▼
   │    ⑥ 复盘   自动取底表 → 按你的北极星判命中/扑
   │       留存归因(开头掉=钩子/中段掉=结构) · 限流预警 · 抖音日报
   │          │
   └──────────┘  闭环:验证好的套路加权回套路库 → 下轮选题更准
```

每一环读什么档案字段、用什么机制,细节见 [工作流.md](工作流.md)。

## 设计原则(为什么它是"通用"的)

1. **引擎读档案,不写死任何人**。同一条数据喂不同档案,复盘判定/选题打分/信息源/视觉风格全都不同——有回归测试守着(`engine/test_judge.py`)。
2. **机制 vs 样本分开**。爆款机制(钩子14招/评分卡框架/2秒跳出标尺)共用;爆款样本(你的赛道弹药)由对标蒸馏种进你自己的套路库。
3. **你随时能改**。北极星/信息源/打分权重/口吻/红线全在你的 `profile.json`,改完有校验兜底(`validate_profile.py`)。
4. **数据反推优于嘴说**。你说想涨粉,但数据显示你的强项是收藏,agent 会把冲突摆出来让你拍板。

## 依赖与降级

| 依赖 | 必需? | 缺了会怎样 |
|---|---|---|
| Claude Code | ✅ | skills 跑不了 |
| Python 3.10+ / venv | ✅ | 取数/判定脚本跑不了 |
| 浏览器登录抖音 | 取数需要 | 完播/涨粉/留存拉不到;可手动存 cookie 到 `~/.douyin_cookie` 兜底 |
| blogger-distiller skill(装到 `~/.claude/skills/`,找作者要) | 推荐 | 对标蒸馏(onboarding 第5问/文案学博主口吻)降级为手动贴对标内容 |
| TikHub Token(第三方付费,blogger-distiller 用) | 可选 | 对标蒸馏/小红书信息源降级为手动贴内容 |
| aihot / last30days skill | 可选 | AI 热点聚合/近30天讨论深挖降级(GitHub/HN 一手源内置,不受影响) |

## 隐私说明

- cookie 只在你本机读取和使用,不上传任何地方。
- 你的真实档案(`profiles/你的id/`)默认**不被 git 追踪**(.gitignore 已配),仓库里只有示例档案。
- 复盘拉到的原始数据 json 同样不提交。

## 已知边界(诚实版)

- 底表取数目前只做了**抖音**;小红书等平台的取数适配待做(判定引擎本身是平台无关的)。
- 抖音红线来自官方创作者学院;小红书红线库为通用运营知识整理。
- 语气没法全自动:想让文案像你,必须喂你自己的旧稿(onboarding 第6问)。
- 日报逐条拉数据,视频多时较慢(平台接口限速)。
