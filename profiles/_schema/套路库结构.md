# 套路库结构（每创作者一份样本库）

> 区分清楚:**机制**在 `共享机制/`(所有人共用),**样本**在 `profiles/{id}/套路库.md`(每人一份,由对标蒸馏种入)。
> 引擎(选题/文案)读机制框架 + 读这个号的样本弹药。新创作者的套路库由 onboarding 第5问对标蒸馏自动种入,不手抄迪迪的。

## 一份 `profiles/{id}/套路库.md` 该有什么

| 段 | 内容 | 来源 |
|---|---|---|
| §0 信号阈值 | 这个号的收藏率/转赞等判精准的口径(抖音0.7/小红书1.0…) | profile.signal_thresholds + 数据基线 |
| §1 选题样本 | 九类公式各填该赛道真实爆款样本(哪类对该号权重高) | 对标蒸馏 + 自家复盘 |
| §2 钩子样本 | 该赛道有效的开头句/锚点 | 对标蒸馏 |
| §3 结构样本 | 该赛道高完播的中段骨架 | 对标蒸馏 |
| §4 金句样本 | 作者真信的判断句 | tone_samples + 自家 |
| §5 CTA样本 | 该号导私域/导咨询的有效话术 | profile.monetization + 蒸馏 |
| §独家点 | = profile.persona.moat,别人抄不走的结合点 | profile |
| §对标拆解卡 | blogger-distiller 产出的拆解卡归档 | 蒸馏 |

> 框架(九类公式/钩子14招/2秒标尺/结构金句CTA)走 `共享机制/`,本文件只装**该赛道的具体弹药**。
> 复盘环节验证好的套路在这里加权/降权(见「复盘」skill 闭环)。

## 种法
1. onboarding 第5问拿到 3 个对标 → 各跑 `blogger-distiller`(走 TikHub) → 出拆解卡。
2. 拆解卡的选题套路/钩子/TOP10结构 → 填进 §1-§5 对应段(人工确认后落库)。
3. 起步可空,发布后由复盘逐条沉淀验证过的套路。
