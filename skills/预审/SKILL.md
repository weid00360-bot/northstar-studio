# 发布前预审（通用）

> 对照迪迪版:那版红线写死"变美标签""割韭菜",平台写死抖音。
> 本版**读 `profiles/{id}/profile.json` 的 `redlines` + `platforms`**,平台红线按主战平台挂对应官方口径。

## 红线 = 三层叠加（逐条扫）
```
通用限流诱因 + 平台模块(按 profile.platforms[primary]) + profile.redlines(该号定位红线)
```
1. **通用层**:读 `共享机制/红线库/通用限流诱因.md`(贴口令/外链/二维码/站外引导/搬运/夸大词)。
2. **平台层**:据 `profile.platforms` 主战平台读 `共享机制/红线库/{平台}.md`——抖音(含"标题英文=搬运雷")、小红书(导流更严/功效词/报备)。官方与通用知识冲突以官方为准。
3. **定位层**:逐条对 `profile.redlines`——迪迪=变美/割韭菜;职场号=虚假offer/包过;带货号=虚假功效,**各按各档案**。

## 🔴 两条最容易漏的硬红线(单独盯)
- **AI 标注**:内容(封面/演示/口播)是 AI 生成的 → 发布必须勾"内容由 AI 生成"声明,否则可能降权限流(细节见平台模块,如 `抖音.md` D段)。口播主动说"这条我用 AI 做的"反而加分。
- **资质门槛**:别滑进无资质领域——医疗诊断/荐药、金融荐股/收益承诺、法律意见、教育升学承诺。讲 AI 蹭到这些必须收住。

## 扫描方法（召回网+防误杀·融自 traffic-compass，别只靠读语境）
1. **词库召回(先兜底,别漏)**:拿 `共享机制/红线库/违禁词库与白名单.md` 的词表把文案/标题/tag 扫一遍,见词先标。
2. **AI 读语境(判真假)**:看上下文是不是真在宣传商品/承诺效果/导流站外。
3. **白名单防误杀(别误伤)**:过白名单,主观表达/序数/科普引用("我最喜欢""第一次")放过;拿不准→标🟡人工看,别直接判🔴。
4. **命中走最小改动**:只换违规那几个词,保作者口语腔和节奏,别改成 AI 腔(见 `共享机制/人味儿写作.md`)。

**🟢 正向检查(通用)**
- 钩子结果先行?结尾有认知金句?有按 `profile.monetization` 的导私域钩子?标题↔钩子呼应?

## 输出
逐条列 🔴必改 / 🟡建议改 / 🟢OK,每条给**最小改动**(只换违规那几个词,保口语腔)。末句总结:可发 / 改完可发 / 不建议发。
