# 红线库（按平台分模块）

> 「预审」skill 据 `profile.platforms[primary]` 挂对应平台红线 + 叠加 `profile.redlines`(该号定位红线)。
> 红线 = 平台特定(抖音≠小红书),所以分模块。**官方口径权威**:能拿到平台官方规则的优先,通用知识标"待官方核"。

## 结构
- `通用限流诱因.md` —— 跨平台共性(贴口令/外链/二维码/站外引导/AI标注/资质门槛),所有平台叠加。
- `违禁词库与白名单.md` —— 召回网词表(广告法极限词/夸大/导流/迷信…)+ 防误杀白名单,预审第1-3步扫描用。
- `抖音.md` —— ✅ 官方源(创作者学院审核规则,含AI生成声明勾选)。
- `小红书.md` —— ⚠️ 通用知识,待官方核(小红书官方社区规范)。
- (待补)`视频号.md` `B站.md` 等。

## 预审怎么用
```
红线 = 通用限流诱因 + 平台模块(按 profile.platforms[primary]) + profile.redlines(该号定位红线)
```
三层叠加逐条扫。平台官方与通用知识冲突时,**以官方为准**。

> 记忆:平台官方规则是权威源,登录态抓不到的靠浏览器读/用户复制后提炼进本库(见抖音创作者学院做法)。
