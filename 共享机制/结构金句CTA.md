# 中段结构 / 金句 / CTA（通用机制）

> 通用引擎层。从迪迪套路库 §3/§4/§5 抽出框架,剥掉具体博主样本。
> 配合 `选题公式与钩子.md`(前段)、`人味儿写作.md`(口吻)。具体素材由各创作者套路库样本供给。

## §3 中段结构（讲明白 = 高完播）

- **概念必配类比/大白话**:抽象概念用生活化类比落地("奶茶店""不用懂公式")。
- **分层/分领域结构化**:把内容拆成 N 层/N 领域,编号推进。风口/拆解题专用:"它到底怎么运作的 N 层/N 步"。
- **真实演示 > 空讲**:录屏、跑流程、演示成品——可视化是高收藏(藏赞0.9+)的根因。
- **动态视觉引导**:讲 PPT 别让图表静着,箭头/高亮框引导视线;穿插真实界面录屏;核心概念配小图标/小动画。静态 PPT 信息门槛高、压完播。
- **方法论口播骨架(长口播套用)**:①0-2秒痛点+受众点名 ②收藏指令前置+评论区口令 ③权威/背书立专业 ④**可视化框架**(编号+命名+能截图,如"周报4原则") ⑤自问自答埋"A还是B还是C"互动 ⑥金句锚点 ⑦量化反差证据 ⑧结尾CTA。可视化框架是高收藏根因。
- **主动消解门槛 = 种"我也能做"**:反复"没门槛/一两天上手",把复杂要求总结成几个关键。可操作感=收藏关键。

## §4 价值升华收尾（被记住+被收藏的关键）

每条爆款都有**认知金句**=作者真信的判断,说白说狠(配 `人味儿写作.md`:从 TA 真实判断长出来,不是 AI 凑对仗)。
> 落地:每条文案结尾必留一句认知金句。

## §5 CTA / 引导互动 / 转化（按 profile.monetization 铺垫）

- **文档/群钩子**:"提示词放文档/群里,点赞收藏关注" → 导私域。
- **开放式**:"希望有人做了反馈我" → 显真诚。
- **引导互动**:埋能引发共鸣的问题/争议点/选边题 → 激活评论区;发布后及时回评。互动率既喂算法又是平台精选标准。
- **结尾必埋"关注利益点"**:把"关注"和"具体好处"挂钩——"关注我,后面每条都拆一个能直接抄的X"——不是空喊关注,是给关注一个理由。⚠️ 若 `north_star` 含涨粉率,这是漏斗第一道,别只顾导私域漏了涨粉。

### 咨询钩子三句（需导咨询/私域的文案 · 元结构）
1. **多讲判断,少讲现象** → 让用户信你的"诊断能力"。开场抛判断/定论,别只罗列现象。
2. **多讲案例,少讲理论** → 让用户信你"解决过问题"。甩真跑通的案例。
3. **多讲方向,少讲方法** → 让用户想解决、且只能找你解决。

⚠️ **第③句据 north_star 改造**:若北极星=收藏率(藏方法=掉收藏),别照搬"藏方法逼咨询"。正确打法:**方法照样给透**(保收藏),把钩子留在"需要长期沉淀才有的判断力/你的知识库/打分标准"上——方法能抄,标准抄不走(锚 `profile.persona.moat` 护城河,不锚"藏干货")。
