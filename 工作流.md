# 通用内容生产 · 全链工作流

> 任何创作者从入驻到发布复盘的完整管线。每一环都读 `profiles/{id}/profile.json`,机制走 `共享机制/`+`engine/`,**没有一处写死迪迪**。

## 全链

```
        ┌─────────────────────────────────────────────────────┐
        │  onboarding(一次性建档)                              │
        │  6问 + 自动抓(engine/baseline.py 反推北极星)         │
        │  → profiles/{id}/profile.json (engine/validate_profile 校验)
        │  → 对标蒸馏种 profiles/{id}/套路库.md                │
        └─────────────────────────────────────────────────────┘
                          │ 档案建好,下面每天跑
                          ▼
   ┌──► ① 选题  skills/选题
   │      读 profile + 共享机制/选题评分卡 + 路由表(engine/resolve_sources)
   │      拉赛道热点 → 九类公式×moat → 5维打分 → 清单(人工选)
   │         │
   │         ▼
   │    ② 文案  skills/文案
   │      读 profile.tone_samples + 共享机制(钩子/结构金句CTA/人味儿写作)
   │      口播稿(对话迭代,定稿才存)→ 发布包(标题/tag/封面/预测卡)
   │         │
   │         ▼
   │    ③ 预审  skills/预审
   │      红线库(通用+平台 by profile.platforms)+ profile.redlines 三层叠加扫
   │         │
   │         ▼
   │    ④ 视频生产  视频生产/(基础库)
   │      按 profile.visual_style 出封面/幻灯片/编排 → 成片发布
   │         │
   │         ▼
   │    ⑤ 复盘  skills/复盘
   │      engine/review.py 按 profile.north_star 判定 + 归因 + 限流预警
   │         │
   └─────────┘  闭环:验证好的套路加权回 套路库 → 下轮选题更准
```

## 每环的"读 profile"对照表

| 环节 | 读 profile 的什么 | 共享机制/引擎 |
|---|---|---|
| onboarding | (生成 profile) | baseline.py / validate_profile.py |
| 选题 | north_star/audience/moat/content_pillars/sources_track/redlines | 选题评分卡 + 选题公式与钩子 + resolve_sources.py |
| 文案 | tone_samples/persona.moat/monetization/redlines | 人味儿写作 + 结构金句CTA + 选题公式与钩子 |
| 预审 | platforms/redlines/monetization | 红线库/(通用+平台) |
| 视频生产 | visual_style(palette/封面/must_face) | 视频生产/模板库 |
| 复盘 | north_star/anti_metrics/signal_thresholds | review.py + judge.py |

## 验证现状
- 三份档案(迪迪/_测试_带货号/职场琳)全过 validate_profile。
- 同一条数据喂不同档案,复盘卡/评分卡判据/信息源/视觉风格全随档案变(见 engine/test_judge.py、scorecard_context.py、resolve_sources.py、test_baseline.py)。
